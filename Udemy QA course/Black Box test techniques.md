* **Equivalence Partitioning**
  Applies if input data can be divided into equal partitions with similar behavior. 
	  <u>Include 2 partitions negative data before and after. </u>

* **Boundary-Value analysis**
   This technique is good when you need exact precise testing of the data or input. There are some cases where you wont have time or need to test specefic data, so Equivalance partitioning will be enough. 
   <u>To use this approach for each data partition there must be clear start and end. </u>
   ![[Pasted image 20260422104813.png]]
   1. Each value belongs only to one partitioning. 
   2. Equivalance part. and BVA can be applied to both output and input values. 
   3. There must be valid and invalid partitions.
