<p align="center"><b>Sequential Models</b></p>
* **Waterfall** - Each stage of developments executes step by step. Easy to implement, but very unflexible. High cost of error and changes, each change/fix must go full loop of development. Testers start work in the end. Rarely used today.
* **V-model** - Waterfall on steroids. More freedom for a tester, as testing starts earlier. For each dev stage there is a testing activity. But still sequential as user will see only the final product.![[Pasted image 20260421130348.png|356]]

<p align="center"><b>Agile development</b></p>
* **Incremental** - is when you develop a software (or not even a software) like building a Lego. Typical Lego builds step by step if it is a robot you will build legs first, than arms, than body, than head. Separately each part is useless, but together it is a functional robot. 
* **Iterative** - is building a soft by creating like a MVP, when your soft at the beginning looks like a prototype with bare minimum but it's sort of functional, ready for a small demo and easier to fix/change. It's like a painting, when first you draw a patern of a person (for example), than colors it, than polishes it with details, shadows and so on.
